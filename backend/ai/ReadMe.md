# Fraud Detection System With Confidence Logic

A machine learning system for detecting fraudulent banking transactions using behavioral analysis and a dual-score risk framework. The project trains and evaluates multiple classification models on two financial transaction datasets — **BankSim** and the **IEEE-CIS Fraud Detection** dataset — and produces both a **Fraud Probability** and a **Detection Confidence Score** for each transaction.

## Project Overview

Traditional fraud detection systems output only a binary label. This project introduces a **dual scoring framework**:

| Score | Description |
|---|---|
| **Fraud Probability** | The likelihood that a transaction is fraudulent, as predicted by a trained classifier |
| **Detection Confidence** | A reliability score based on behavioral anomaly signals (unusual amounts, missing device info, irregular transaction hours) |

By combining these two scores into a **Final Risk Score**, the system distinguishes between isolated anomalies and persistent suspicious behavior, improving interpretability for fraud analysts.

**Models trained:**
- Logistic Regression
- Random Forest Classifier
- Gradient Boosting Classifier

**Key techniques used:**
- Behavior-based feature engineering (spending statistics, merchant novelty, transaction frequency)
- SMOTE oversampling to handle severe class imbalance
- Standard scaling of numerical features
- ROC-AUC, Precision-Recall, and Confusion Matrix evaluation

## Dataset

This project uses three dataset files spanning two distinct fraud detection datasets.

### 1. BankSim — `bs140513_032310.zip`

A synthetic dataset generated by an agent-based banking payment simulator. It models realistic customer purchasing behavior and injects fraudulent transactions to simulate real-world fraud scenarios.

- ~594,000 transactions over a simulated 6-month period
- ~7,200 fraudulent transactions (~1.2% fraud rate)
- Features include: customer ID, age group, gender, merchant, transaction category, amount, and fraud label

This dataset is used for the initial EDA, feature engineering, and training of the baseline fraud detection models (Logistic Regression, Random Forest, Gradient Boosting).

### 2. IEEE-CIS Transactions — `train_transaction.zip`

A real-world anonymized transaction dataset released as part of a Kaggle competition, provided by Vesta Corporation and IEEE.

- ~590,000 online transactions
- Over 430 anonymized features including card attributes, billing address, and product codes
- Target label: `isFraud`

### 3. IEEE-CIS Identity — `train_identity.zip`

Device, browser, and identity-related attributes associated with the IEEE-CIS transactions.

- Contains device type, device info, and anonymized identity/network features (`id_12` – `id_38`)
- Linked to transaction records via the `TransactionID` field

The two IEEE-CIS files are **merged on `TransactionID`** to create a unified dataset for the advanced modeling phase, which introduces the confidence scoring framework.

- **Train/Test Split:** 80% training, 20% testing

- **Feature Scaling:** `StandardScaler` is applied after SMOTE to normalize numerical features

## Prediction

Predictions are generated directly within the notebook after the models are trained. There is no separate prediction script — all inference logic is contained in the notebook cells.

### Fraud Probability

Each trained model produces a fraud probability score using `predict_proba`. For example, the Gradient Boosting model is used for the final scoring:

```python
# Cell: Fraud Probability using Trained Model
fraud_probability = gb_model.predict_proba(X_test)[:, 1]
```

This outputs a value between 0 and 1 for each transaction, representing the model's estimated probability that it is fraudulent.

### Detection Confidence Score

A behavioral confidence score is computed using three anomaly signals:

```python
# Cell: Confidence Score Calculation
w_amount = 0.4   # weight for unusual transaction amount
w_device = 0.3   # weight for missing device information
w_time   = 0.3   # weight for transaction at unusual hours

df_ieee_clean["confidence_score"] = (
    w_amount * df_ieee_clean["amount_anomaly"]
    + w_device * df_ieee_clean["device_anomaly"]
    + w_time   * df_ieee_clean["time_anomaly"]
)
```

### Final Risk Score

The fraud probability and confidence score are combined into a single risk score:

```python
# Cell: Final Risk Score
confidence = df_ieee_clean["confidence_score"].reindex(X_test_index)
final_risk_score = fraud_probability * confidence
```

### Interpreting Outputs

| Fraud Probability | Confidence Score | Interpretation |
|---|---|---|
| High | High | Strong, reliable fraud signal |
| High | Low | Suspicious but uncertain — warrants review |
| Low | High | Consistent normal behavior |
| Low | Low | Weak signals overall — low risk |

To generate predictions on new data, replace `X_test` with a preprocessed DataFrame that follows the same feature engineering pipeline used during training, then execute the prediction cells.
